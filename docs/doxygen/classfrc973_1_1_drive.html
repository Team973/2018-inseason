Detailed Description

Drive provides an interface to control the drive-base (to do both teleoperated and autonomous movements). To do this, it makes a bunch of DriveControllers (autonomous PID, autonomous trap, teleop arcade, maybe someday a state space drive controller). When a command is issued (one of these routines is called), Drive determines which controller is best suited to service that command and makes it the "active" controller.

DriveBase: calls on the active controller to calculate motor output. DriveStateProvider: provides the controller with position/angle/speed etc. DrivecontrolSignalReceiver: translates controller output signal to motor input signal.
